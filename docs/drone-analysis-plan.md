## 一、算法 pipeline（结合你给的约束）

在「100 米高空无人机 + 4K 视频」场景下，可以先定一个非常清晰的 pipeline：

1. 输入裁剪与时长限制
   - 用户给你一个完整视频，但 DEMO 只取前 30 秒。
   - 后端预处理：读取视频元信息（fps），截出前 30 秒，再做后续处理。

2. 抽帧策略
   - 抽帧为 1 fps：从前 30 秒中抽取约 30 帧，保证计算量可控且仍能看出识别效果。
   - 每帧单独处理，然后把结果汇总成全视频的统计。

3. 4K 分块检测
   - 假设原始分辨率 3840×2160 或类似，YOLOv3 模型输入仍然是 416/608 等尺寸，不能直接喂 4K。
   - 对每一帧，做如下处理：
     - 将 4K 帧按网格切成多个 tile，例如 960×960 或 1280×720，tile 之间适当重叠（比如 10%）防止边缘漏检。
     - 对每个 tile 缩放到模型输入尺寸（如 416×416）做检测。
     - 把每个 tile 的检测结果坐标还原回整张 4K 图的坐标系，然后做一次全帧级的 NMS 合并。
   - 这样可以保证电线杆、光纤标识等小目标在 tile 中像素不至于太小，保持检出率。

4. 特定类别与计数
   - 类似该仓库现有的「指定类别过滤 + 过线计数」逻辑，你只需要：
     - 在 utils 中改成只保留「电线杆 / 铁塔 / 光纤标识」这些自定义类别（对应你训练的模型类别 ID）。
     - 不一定要用「过线计数」，可以更简单：对每帧上每个目标计 1，然后按视频维度做去重（可根据空间位置和时间邻近做简单匹配，避免一根杆在多帧重复计数过多）。
   - DEMO 阶段可以先采用「粗略计数」：直接累加 detections 数量，并说明这是「估计数量」。

5. 输出产物
   - 一段带框标注的短视频（只由前 30 秒抽帧再合成，帧率可以是 5–10 fps 方便观看）。
   - 一份结构化统计结果（JSON 或数据库记录），包含：
     - 总共检测到的电线杆数量、铁塔数量、光纤标识数量；
     - 每类在每帧的计数数组（用于前端绘制简单条形图或折线图）。

## 二、如何基于现有仓库改造

你参考的项目已经有 YOLOv3 视频 demo 和计数逻辑，可以最小改动实现你的 pipeline：

1. 利用 video_demo.py 的基础框架
   - 保留视频读取、模型加载、逐帧推理、绘制框和保存结果视频的逻辑。
   - 在读取视频时：
     - 记录 fps、总帧数，计算前 30 秒对应的最大帧 index（min(帧率 * 30, 总帧数)）。
     - 在主循环里，只处理到这个帧 index，后面的直接 break。

2. 在每帧处理函数中加入分块流程
   - 把原来「resize 到模型输入 → 直接检测」改为：
     - 先按 tile grid 划分 4K 帧（比如 4×2 = 8 个 tile），对每个 tile 单独调用检测器。
     - 将每个 tile 的检测框坐标换算回原始帧，再做一次 NMS 合并。
   - 这一层逻辑可以写在 utils 里，保持 video_demo.py 尽量简单。

3. 统计逻辑替换
   - 当前代码里有「指定类别过滤、画线计车 / 行人」等逻辑，可以改为：
     - 使用指定类别 ID 列表（杆 / 塔 / 标识），过滤出目标。
     - 对当前帧的 detections：
       - 按类别计数，累加到一个全局计数器。
   - 如果要减轻重复计数，可以：
     - 简单用 IOU + 类别对比，把相邻帧匹配上的目标看作同一目标，不重复计数；
     - 原型阶段也可以直接总数叠加，页面上写清楚「为便于展示，统计不做严格去重」。

4. 结果输出给前端
   - 在视频处理结束时，将统计数据保存为 JSON 文件或写入数据库，例如：
     - video_id
     - total_poles, total_towers, total_fiber_signs
     - per_frame_counts（可选）
   - 一页式前端只需要用一个简单接口（如 /api/video_stats?video_id=xxx）拉取这段 JSON。

## 三、一页式前端交互设计（在你「不让用户自己上传」前提下）

因为你说「用户上传先不管，我们手工上传到后台」，可以把前端变得更简单：

1. 页面主要模块

- 模块 1：视频选择
  - 提供一个下拉框/卡片列表，列出几段已经处理好的无人机 4K 视频（你内部手动跑好）。
  - 用户点击某个视频后，前端向后端请求该 video_id 的统计数据和结果视频 URL。

- 模块 2：统计数据展示
  - 显示：
    - 电线杆：N 个
    - 铁塔：M 个
    - 光纤标识：K 个
  - 下方用一个简单条形图或饼图展示各类别占比（可以用 ECharts/Chart.js 实现）。

- 模块 3：结果视频区域
  - 放一个 video 元素播放「前 30 秒处理后的回放」，画面里有边框和类别标签。
  - 可选放一张代表性帧图片（screenshot）以减少加载压力。

2. 页面上的约束说明

在页面上明确标出 DEMO 限制，管理预期：

- 只展示视频前 30 秒分析结果。
- 抽帧频率为 1 fps，以演示算法能力，不代表最终部署性能。
- 当前结果在 100 米高度的无人机拍摄条件下优化，其他场景可定制。
